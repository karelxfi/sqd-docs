## Core Capabilities

Portal provides four key features that make it ideal for data pipelines:

<div className="space-y-6 my-8">
  <div className="p-6 rounded-xl bg-white dark:bg-gray-900 border border-slate-200 dark:border-slate-700">
    <h3 className="font-bold text-xl mb-3 text-gray-900 dark:text-white">
      1. Arbitrary Block Ranges
    </h3>
    <p className="text-gray-700 dark:text-gray-400 mb-3">
      Request any block range in a single HTTP POST—no manual pagination through 
      individual blocks. Essential for backfills and historical analysis.
    </p>
    <div className="bg-slate-50 dark:bg-slate-900/50 p-4 rounded-lg">
      <code className="text-sm">
        POST /stream with fromBlock: 18000000, toBlock: 18100000
      </code>
    </div>
  </div>

{" "}

<div className="p-6 rounded-xl bg-white dark:bg-gray-900 border border-slate-200 dark:border-slate-700">
  <h3 className="font-bold text-xl mb-3 text-gray-900 dark:text-white">
    2. Native Finalization
  </h3>
  <p className="text-gray-700 dark:text-gray-400 mb-3">
    Built-in reorg handling with chain-specific finality rules. Cleanly
    communicates rollbacks as structured events in the stream.
  </p>
  <div className="bg-slate-50 dark:bg-slate-900/50 p-4 rounded-lg">
    <code className="text-sm">
      Use /finalized-stream for production-safe data ingestion

</div>

{" "}

<div className="p-6 rounded-xl bg-white dark:bg-gray-900 border border-slate-200 dark:border-slate-700">

    3. HTTP Streaming
  </h3>
  <p className="text-gray-700 dark:text-gray-400 mb-3">

    of massive ranges. Process terabytes without loading everything into RAM.
