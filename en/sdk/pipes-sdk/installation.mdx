# Installation

This guide covers installing Pipes SDK and setting up your development environment.

## Package Installation

Pipes SDK requires two main packages:

- `@sqd-pipes/pipes` - Core pipeline functionality
- `@sqd-pipes/evm-processor` - EVM blockchain data source

<Steps>
<Step title="Install core packages">

<CodeGroup>
```bash npm
npm install @sqd-pipes/pipes @sqd-pipes/evm-processor
```

```bash yarn
yarn add @sqd-pipes/pipes @sqd-pipes/evm-processor
```

```bash bun
bun add @sqd-pipes/pipes @sqd-pipes/evm-processor
```

</CodeGroup>

</Step>

<Step title="Install TypeScript dependencies">

Pipes SDK is written in TypeScript and requires TypeScript to be installed:

<CodeGroup>
```bash npm
npm install -D typescript ts-node @types/node
```

```bash yarn
yarn add -D typescript ts-node @types/node
```

```bash bun
bun add -D typescript @types/node
```

</CodeGroup>

<Note>
  If you're using Bun, you can run TypeScript files directly without ts-node.
</Note>

</Step>

<Step title="Initialize TypeScript configuration">

Create a `tsconfig.json` file:

```bash
npx tsc --init
```

Or create it manually with recommended settings:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "outDir": "./lib",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

</Step>
</Steps>

| yarn            | ✅        | Full support                        |

## Optional Dependencies

### For Clickhouse Integration

If you plan to use Clickhouse as a data target:

<CodeGroup>
```bash npm
npm install @clickhouse/client
```

```bash yarn
yarn add @clickhouse/client
```

```bash bun
bun add @clickhouse/client
```

</CodeGroup>

### For ABI Type Generation

To generate TypeScript types from contract ABIs:

<CodeGroup>
```bash npm
npm install -D @subsquid/evm-typegen
```

```bash yarn
yarn add -D @subsquid/evm-typegen
```

```bash bun
bun add -D @subsquid/evm-typegen
```

</CodeGroup>

## Project Structure

A typical Pipes SDK project structure:

```
my-pipeline/
├── src/
│   ├── index.ts          # Main pipeline code
│   ├── abi/              # Generated ABI types (optional)
│   └── transformers/     # Custom transformers (optional)
├── package.json
├── tsconfig.json
└── docker-compose.yml    # For local Clickhouse (optional)
```

## Verify Installation

Create a simple test file to verify your installation:

```typescript test.ts
import { createEvmSource } from "@sqd-pipes/evm-processor";

const source = createEvmSource({
  gateway: "https://v2.archive.subsquid.io/network/ethereum-mainnet",
  chain: "ethereum-mainnet",
  query: {
    range: { from: 20_000_000, to: 20_000_000 },
  },
});

console.log("Pipes SDK installed successfully!");
```

Run it:

<CodeGroup>
```bash npm
npx ts-node test.ts
```

```bash bun
bun test.ts
```

</CodeGroup>

<Check>
  If you see "Pipes SDK installed successfully!" your installation is complete.
</Check>
