# SDK overview

**TypeScript libraries that transform raw Portal data into actionable insights**

Our SDK provides type-safe TypeScript libraries for building blockchain data pipelines. While Portal gives you raw blockchain data, the SDK adds data transformation, decoding, and persistence capabilities on top of it.

## How SDK Relates to Portal

<Frame>
  <img
    src="/images/portal-overview.gif"
    alt="Portal data flow showing raw blockchain data being transformed by SDK and stored in your database"
  />
</Frame>

- **Portal** provides the raw blockchain data through HTTP API
- **SDK** uses Portal as its data source and adds:
  - Event and transaction decoding
  - Type-safe data transformation
  - Batch processing capabilities
  - Database persistence
  - Real-time data ingestion

<Info>
  SDK uses Portal under the hood. When you use SDK, you're still accessing
  blockchain data through Portalâ€”the SDK just makes it easier to work with.
</Info>

## Key Benefits

- **Type Safety** - Full TypeScript support with auto-generated types from ABIs
- **Automatic Decoding** - Decode logs, transactions, and state changes automatically
- **Flexible Persistence** - Write to PostgreSQL, ClickHouse, files, or custom targets
- **High Performance** - Optimized batch processing with backpressure handling
- **Developer Experience** - Hot reload, local development, comprehensive tooling

## Choosing the Right Tool

Use **SDKs** when you need automatic ABI decoding, data transformation, database persistence, or type-safe development. Use **Portal** directly for non-TypeScript languages, maximum HTTP control, or simple data extraction. For detailed comparisons between Portal, Pipes SDK, and Squid SDK, see our [comparison guide](/en/sdk/comparison).
